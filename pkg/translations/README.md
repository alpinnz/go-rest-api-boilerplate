# Translation Module Guide

## 1. Folder Structure

```
locales/
├── en/
│   ├── user.json
│   └── role.json
├── id/
│   ├── user.json
│   └── role.json
internal/
└── translations/
    ├── store.go        # Translation store
    └── constants.go    # Generated constants from JSON keys
```

* Each JSON file contains keys for a specific module.
* Keys can use dot notation, e.g., `"user.create_success"` or `"role.not_found"`.
* Keys from all modules per language are merged into `translations.Store`.

---

## 2. JSON Example

**`locales/en/user.json`**

```json
{
  "user.create_success": "User created successfully",
  "user.not_found": "User not found",
  "user.welcome": "Welcome, {{.Name}}!"
}
```

**`locales/id/user.json`**

```json
{
  "user.create_success": "Pengguna berhasil dibuat",
  "user.not_found": "Pengguna tidak ditemukan",
  "user.welcome": "Selamat datang, {{.Name}}!"
}
```

* `{{.Name}}` is a template placeholder replaced dynamically using a map.

---

## 3. Generated Constants

After running `generate_translations.go`:

```go
package translations

// Code generated by generate_translations.go; DO NOT EDIT.

const (
	ROLE_NOT_FOUND      = "role.not_found"
	USER_CREATE_SUCCESS = "user.create_success"
	USER_NOT_FOUND      = "user.not_found"
	USER_WELCOME        = "user.welcome"
)
```

* Use constants to avoid typos in controllers or services.

---

## 4. Load Translations in Code

```go
import "internal/translations"

tr := translations.New()

// Load all modules from locales folder
if err := tr.LoadAllModules("locales"); err != nil {
	log.Fatal(err)
}
```

---

## 5. Translate a Message

```go
lang := "id" // or get from request header

// Simple translation without template
msg := tr.T(lang, translations.USER_NOT_FOUND, nil)
fmt.Println(msg) // Output: "Pengguna tidak ditemukan"

// Translation with template parameters
msg2 := tr.T(lang, translations.USER_WELCOME, map[string]any{"Name": "Alfin"})
fmt.Println(msg2) // Output: "Selamat datang, Alfin!"
```

---

## 6. Fallback Mechanism

* If key is missing in the requested language, it falls back to English ("en").
* If key does not exist in English either, it returns the key string itself:

```go
msg := tr.T("fr", translations.USER_CREATE_SUCCESS, nil) // fallback to English
```

---

## 7. Recommended Usage in Controller

```go
func (h *UserController) CreateUser(c *gin.Context) {
	lang := middleware.GetLang(c, "en_EN") // fallback to "en_EN"
	tr := middleware.GetTranslations(c)

	user, err := h.userUsecase.Create(c.Request.Context(), req)
	if err != nil {
		c.JSON(400, gin.H{
			"message": tr.T(lang, translations.USER_NOT_FOUND, nil),
		})
		return
	}

	c.JSON(200, gin.H{
		"message": tr.T(lang, translations.USER_CREATE_SUCCESS, map[string]any{"Name": user.Name}),
	})
}
```

---

## 8. Template Parameters (Map)

* Use `map[string]any` for template substitution.

Example JSON:

```json
"user.welcome": "Welcome, {{.Name}}!"
```

Example Go:

```go
tr.T("id", translations.USER_WELCOME, map[string]any{"Name": "Alfin"})
// Output: "Selamat datang, Alfin!"
```

* Multiple parameters supported:

```go
tr.T("en_EN", translations.SOME_KEY, map[string]any{"Name": "Alfin", "Role": "Admin"})
```

## 9. Summary

| Feature                | Usage / Notes                                     |
| ---------------------- | ------------------------------------------------- |
| Modular JSON           | `locales/<lang>/<module>.json`                    |
| Auto merge             | All keys merged per language                      |
| Constants              | Generated via script → avoids typos               |
| Template support       | `{{.Param}}` in JSON, replaced by map\[string]any |
| Fallback               | Missing key → English → returns key string        |
| Controller integration | Use `middleware.GetTranslations(c)` and `GetLang` |
